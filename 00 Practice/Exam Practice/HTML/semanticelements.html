Semantic Element

Semantic elements are elements with meaning. The elements like div, span are non-semantic elements as they tells nothing about the content whereas the semantic elements clearly describes its meaning.

The html semantic elements are:

  1. <article>
  2. <aside>
  3. <details>
  4. <figcaption>
  5. <figure>
  6. <footer>
  7. <header>
  8. <main>
  9. <mark>
  10. <nav>
  11. <section>
  12. <summary>
  13. <time>

Main Element

The main element defines the main content of a document. The content inside a main element must be unique throught a document. It should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logo, and search forms. There must not be more than one main element. Main element must not be a descendent of an article, aside, footer, header, or nav element.

Section Element

The <section> element is used to define the section in an html document. A section is a thematic grouping of content, typically with heading. The section elements can be used in following places:

  • Chapters
  • Introductions
  • News Items
  • Contact Information

A web page could normally be split into sections for Introduction, content and contact information.

Article Element

The article element is an independent, self-contained content. An article should make sense on its own. Examples where the articles elment can be used:

  • Forum Posts
  • Blog Posts
  • User Comments
  • Product Cards
  • Newspaper Article

Can we nest <section> and <article>?

Yes, we can nest them within each other. This flexibility allows to create complex and structured content.

Aside Element

A aside element defines some content aside from the content it is placed in (like sidebar). A aside content should be indirectly related to the surrounding content.

Header Element

The header element is a container for introductory content or set of navigation links.

A header element typically consists:

  • one or more heading elements like <h1>-<h6>
  • logo or icon
  • authorship information

We can have several header elments in one html document. However, header cannot be placed within a footer, address or another header element.

Figure and Figcaption Element

The figure tag specifies self-contained content, like photo, diagrams etc. The figcaption tag defines a caption for figure element.

Footer Element

Footer defines a footer of a document or section. A footer element typically contains:

  • Authorship Information
  • ©Copyright Information
  • ☎Contact Information
  • Sitemaps
  • Back to top links
  • Related Documents

We can have several footer element in a document.
